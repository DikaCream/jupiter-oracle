The Core Thesis

What if prediction markets are the ultimate trading oracle?

01 // The Signal

We use Jupiter's Prediction API (bridging Polymarket/Kalshi liquidity) to read live odds. If the market says there is a 75% chance BTC hits $150k, that is our oracle signal.

02 // The Action

Instead of just plotting charts, we arm a Sentinel. When the odds hit a user-defined threshold, it automatically builds and triggers a real on-chain swap via Jupiter Swap API.

Architecture

100% Client-Side. 0 Middleware. 5 Jupiter APIs.

[ Browser / React App ] │ ├── 1. GET /prediction/v1/events (Poll every 10s) │ ↳ Populates live markets table & triggers Sentinel check │ ├── 2. GET /prediction/v1/orderbook (Real-time depth) │ ├── 3. GET /price/v3 (Live prices for SOL/USDC) │ └── 4. Sentinel Threshold Reached? ↳ YES: GET /swap/v2/order ↳ Returns Base64 Tx → Phantom Wallet → Solana Mainnet

The Sentinel Engine

Executing swaps based on probability thresholds.

Automated Execution

The user sets a YES% threshold (e.g., 80%). The app polls Jupiter APIs every 10s. The moment the live odds cross 80%, the frontend requests an optimally routed swap transaction.

Native Integration

Using window.solana to inject the unsigned transaction directly into Phantom. No complex backend or RPC node required. Pure REST API to on-chain execution.
